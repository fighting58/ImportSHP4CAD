# AutoCAD C# 텍스트-다중지시선(MLeader) 변환 플러그인 개발을 위한 완벽한 프롬프트

이 문서는 AutoCAD .NET API(C#) 환경에서 **기존 단일행/다중행 텍스트를 정밀한 다중 지시선(MLeader) 객체로 자동 변환 및 정렬 배치하는 기능**을 구현하기 위한 최종 개발 프롬프트 가이드입니다.

사용자가 도면 상의 기존 텍스트를 클릭(1번 점)하고 화살표의 진행 방향 끝점(2번 점)을 지정하면, 텍스트의 기하학적 정중앙에서 시작하여 클릭한 끝점까지 완벽하게 정렬된 수평 지시선(MLeader)을 생성하는 고급 기하 오프셋 수식이 적용되어 있습니다.

## 🚀 AI 코딩 에이전트용 복사 가능 프롬프트 템플릿

```markdown
# [Task] AutoCAD .NET C# 텍스트-MLeader 정밀 변환 및 정렬 명령어 개발

## 1. 역할 및 목표
당신은 AutoCAD .NET Framework API(C#) 환경에서 작동하는 도면 자동화 플러그인 전문가입니다.
도면 내의 기존 텍스트(DBText 또는 MText)를 선택하고 지시선이 뻗어 나갈 방향 점(2번 점)을 클릭받아, 기존 텍스트를 제거하는 동시에 텍스트 내용/크기/레이어를 그대로 승계하여 화살촉 2.0, 수평 지시선(Dogleg) 2.0, 문자 정렬 중간(MiddleCenter) 스펙의 정밀한 다중 지시선(MLeader)을 생성하는 플러그인을 개발하세요.
명령어의 단축 실행 키는 `TEXT2LEAD` 및 `T2L`로 정의합니다.

---

## 2. 세부 기하학적 매핑 규칙 및 오차 보정 공식 (수식 준수 필수)

### [정확한 1번/2번 기준점 획득]
1. **1번 위치점 (원본 텍스트의 기하학적 중심)**:
   * 입력받은 텍스트의 원래 각도를 강제로 0으로 보정합니다 (`Rotation = 0`).
   * 텍스트의 정적인 단순 기점(Position/Location) 대신, 변경된 회전상태를 그래픽스 엔진에 강제 반영(`tr.TransactionManager.QueueForGraphicsFlush()`)한 뒤 객체의 **`GeometricExtents`** 경계 상자를 조회합니다.
   * 경계 영역의 좌하단($Min$)과 우상단($Max$) 좌표의 산술평균을 통해 텍스트 본연의 **완벽한 정중앙 중심점 $startPt$**를 산출하여 1번 점으로 삼습니다:
     $$startPt = \left(\frac{Min.X + Max.X}{2.0}, \frac{Min.Y + Max.Y}{2.0}, \frac{Min.Z + Max.Z}{2.0}\right)$$
   * 텍스트의 가로 폭 $textWidth = Max.X - Min.X$를 함께 구해둡니다.
2. **2번 위치점 (사용자가 두 번째 클릭한 지시선 끝점)**:
   * 사용자 화면 클릭 입력값 $endPt$를 2번 점으로 정의합니다.

---

### [지시선(MLeader) 기하학 정렬 및 오프셋 보정 공식]

지시선의 기하 구성 정점과 문자의 배치 삽입점($TextLocation$)은 다음과 같이 삼각 오차 보정식에 의해 완벽하게 정렬되어야 합니다.

1. **지시선 정점 배정**:
   * **화살표 팁 (`FirstVertex`)**: 1번 위치점 ($startPt$, 원본 텍스트의 기하 중심)
   * **지시선 꺾임점 (`LastVertex`, Shoulder)**: 2번 위치점 ($endPt$)
2. **수평 랜딩(Dogleg) 방향 결정**:
   * $endPt.X < startPt.X$ 인 경우 좌측 수평 방향 벡터 적용: $landingDir = -Vector3d.XAxis$
   * 그 외의 경우 우측 수평 방향 벡터 적용: $landingDir = Vector3d.XAxis$
3. **MText 겹침 방지 및 Y축 시프트(처짐) 상쇄 보정 ($TextLocation$ 산출)**:
   MLeader의 텍스트 자리맞춤이 `MiddleCenter`인 경우, AutoCAD 내부 렌더링 엔진은 텍스트 전체 무게중심을 기준으로 Y축 방향으로 텍스트 높이의 절반($textHeight / 2.0$) 만큼 지시선 어깨점의 Y좌표를 깎아내려 정렬합니다. 
   이로 인해 지시선 꺾임선이 입력한 높이보다 밑으로 시프트되는 것을 방지하고 텍스트가 지시선과 겹치지 않게 하려면 다음과 같은 정밀 오프셋 보정 수식을 사용해 $TextLocation$을 구해야 합니다:
   
   * **X축 오프셋 보정 ($xOffset$)**: 문자의 중간정렬 기점을 감안하여 랜딩선과 문자가 겹치지 않도록 가로 폭의 절반을 더합니다.
     $$xOffset = landingDir \times \left(DoglegLength + LandingGap + \frac{textWidth}{2.0}\right)$$
   * **Y축 오프셋 보정 ($yOffset$)**: 내부 엔진의 처짐 렌더링 오차를 완전히 상쇄하여 꺾임점을 원래 Y클릭 좌표에 수평 고정합니다.
     $$yOffset = Vector3d.YAxis \times \left(\frac{textHeight}{2.0}\right)$$
   * **최종 텍스트 위치 공식**:
     $$TextLocation = endPt + xOffset + yOffset$$

---

## 3. 사용자 인터페이스 (Editor Prompts) 및 안전장치
* **스냅 간섭 방지**: 사용자로부터 2번 끝점을 입력받기 전에 현재 캐드의 스냅 설정 시스템 변수인 `OSMODE`를 백업해두고, `OSMODE = 0`으로 설정하여 마우스 스냅 스냅 간섭 없이 사용자가 원하는 화면 상의 정밀 좌표를 획득합니다. 작업 종료(성공/실패/취소 모두 포함) 시에는 반드시 원래 `OSMODE`로 복원합니다.
* **Undo 지원 및 예외 처리**: 작업 실패 또는 사용자의 취소 시 회전각이 0으로 누운 기존 텍스트가 원래대로 원상태 복구되도록 Transaction Abort를 구성하고, 성공 시에만 Commit하여 완벽한 원상태 회복 성능을 구현합니다.
* **기존 엔티티 처리**: 지시선 변환 생성이 완료된 직후, 기존의 원본 텍스트 엔티티는 도면 공간에서 완벽히 삭제(`Erase(true)`) 처리합니다.
```

## 💡 이 프롬프트를 통해 기대되는 구현 성과
위 프롬프트는 AutoCAD 지시선 설계 시 현업에서 흔히 마주치는 기하학적 난제를 완벽하게 해소합니다:
* **완벽한 수평 고정**: AutoCAD 엔진의 특성에 의해 지시선 꺾임점이 y축 아래로 처지던 고질적인 버그를 수학적 오프셋 가산 기법($yOffset$)으로 완벽하게 상쇄하여 고정합니다.
* **텍스트 겹침 완전 해결**: 중간정렬 상태에서도 문자 가로 폭의 절반($textWidth/2.0$)만큼 정적 이동 연산을 연동하여, 도면 상에서 글씨가 지시선 꼬리선에 겹쳐 가려지는 현상을 근본적으로 차단합니다.
