ImportSHP4CAD

GIS 공간 데이터와 CAD 도면 편집의 고속화를 실현하는 AutoCAD 전용 자동화 솔루션

프로젝트 시스템 개요

ImportSHP4CAD는 AutoCAD 2013 .NET API를 기반으로 제작된 정밀 플러그인 라이브러리입니다. GIS 공간 레이어(.shp)의 기하 형상(Geometry)과 속성값(Object Data)을 도면에 깨짐 없이 고속 로드하고, 도곽선 획정, 일련번호 연속 기입, 다층 구조의 정점 편집 및 면적 미세 조정을 극대화된 사용성(UX)으로 신속하게 처리할 수 있도록 개발되었습니다.

Windows 11 및 기존 구버전 캐드 플랫폼과의 멀티스레딩/메모리 충돌 프리징 방지 설계를 탑재하여 높은 안정성을 보장합니다.

[시스템 환경 규격]

• Target SDK: AutoCAD 2013 .NET API
• OS 호환성: Windows 11 / 10 x64
• 기반 라이브러리: NTS 1.13.2
• 문자 스타일: SHP_한글 (Malgun Gothic)

신규 릴리즈 핵심 보안 및 한글 패치 기능 상세

1. ISH 명령어 인코딩 & 한글 깨짐 원천 방지 (DbfRawReader 직접 해독)

EUC-KR & UTF-8 호환

NTS(NetTopologySuite) 라이브러리가 지닌 디코딩 초기화 한계(DbaseFileReader 생성 단계에서 오인코딩하여 이미 깨진 채로 읽어들이는 한계)를 완벽하게 배제하기 위하여, 파일 스트림의 바이트 배열을 물리적으로 직접 스캔하는 자체 바이너리 파서(DbfRawReader)가 적용되었습니다.

[지능형 다단계 감지 알고리즘 작동 프로세스]

  • CPG 명시적 매핑: 불러온 파일 명과 동일한 위치에 .cpg 파일이 있는 경우, 우선적으로 내용을 해석하여 EUC-KR(949) 혹은 UTF-8로 지정합니다.
  • 바이트 시퀀스 휴리스틱 검사: CPG 파일이 없더라도 DBF 본문 데이터(앞쪽 5개 행)의 바이트 패턴을 자동 분석하여, 유효한 UTF-8 형식의 멀티바이트 글자 패턴이 충족되는 경우 자동으로 UTF-8로 간주하여 디코딩을 수행합니다.
  • 안전장치(Fallback): 식별되지 않거나 일반적인 로컬 GIS 가공 데이터의 경우 EUC-KR(949)을 기본값으로 해독합니다.

[한글 폰트 매핑 스타일 지원]

AutoCAD 도면의 표준(Standard) 문자 스타일이 영문 글꼴로 설정되어 있더라도, 플러그인이 도면 공간에 윈도우 시스템 한글 폰트(맑은 고딕: malgun.ttf)가 연동된 SHP_한글 스타일을 자동으로 확보하여 DBText 객체에 바인딩하므로 화면 상의 폰트 깨짐이 존재하지 않습니다.

2. INCNUM 명령어의 대화상자 인터페이스 및 연속 자동 증분

Win11 DWM 프리징 방지

기존 캐드 플러그인에서 모달 대화상자 호출 시 Windows 11 Desktop Window Manager(DWM)와의 초점 불일치로 인하여 발생하던 캐드 먹통(Freezing) 현상을 완벽 차단하였습니다.

[핵심 개선점 및 동작 순서]

  • MainWindow 핸들 결합: Application.ShowModalDialog(Application.MainWindow.Handle, form) 호출 방식을 사용하여 모달 창을 띄울 때 소유자 핸들을 강제 고정함으로써 포커스 이탈에 따른 DWM 프리징이 발생하지 않습니다.
  • 연속 자동 증분: 단일 클릭 루프를 기입하고 종료하면, 기입이 종료된 마지막 번호의 다음 번호를 내부 스태틱 메모리에 자동 보관합니다. 재실행 시 다이얼로그 폼의 시작 번호 텍스트박스에 해당 번호가 기본값으로 자동 매핑되어 마우스 클릭만으로 작업을 연속할 수 있습니다.
// [동작 과정 안내]
// 1. 커맨드 입력: INCNUM
// 2. 대화상자 팝업: 시작번호(예: 101), 글자크기(예: 0.80), 접두사, 접미사(Suffix) 입력
// 3. 도면 임의 지점 연속 클릭 -> 클릭할 때마다 101, 102, 103... 문자 생성
// 4. 완료 후 재실행 시, 자동으로 시작 번호에 104가 기본 제안되어 수동 값 수정 없이 바로 연속 기입 가능
